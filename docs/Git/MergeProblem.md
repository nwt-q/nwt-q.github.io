# Git 合并存在的问题

**场景还原：**

在多人协助的版本管理中，还是建立不同分支合并效果会更好些，因为如果使用同一分支，A成员提交后B成员修改后尝试 push 会出现如图所示的问题

![本地仓库与远程仓库不匹配](assets/image-20251219101743889.png)

**本地仓库与远程仓库不匹配**



如果B成员尝试 pull 则会出现以下问题

1. A成员修改的内容被B同学修改的内容影响，出现差异需要手动修改
2. A同学与B同学都修改不同的内容互不影响
3. A同学与B同学文件名称一样内部内容却不一样，造成pull失败需要手动修改

总之使用一个分支会造成版本与版本间存在差异的问题，应当谨慎使用，或者创建新的分支，组长以合并的方式解决问题
