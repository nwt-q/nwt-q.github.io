# 局域组网笔记

## 一、局域网（LAN）基础概念

局域网是指在某一区域内（通常是1km范围内）由多台计算机、终端设备、网络设备（路由器、交换机等）通过传输介质连接组成的封闭网络，核心特点是*范围小、速度快、延迟低、安全性高*，主要用于企业、家庭、校园等局部场景，实现设备间的文件共享、打印机共享、网络访问等功能。

核心作用：解决局部范围内多设备互联互通，避免单设备孤立，提升办公、生活的便捷性，同时可通过路由器连接外网，实现多设备共享网络。

## 二、局域组网核心设备

组网需根据场景（家庭/小型办公/大型企业）选择设备，核心设备及功能如下：

- **路由器**：核心设备，负责连接外网（宽带）和内网设备，分配IP地址，实现内网设备与外网、内网设备之间的通信，部分路由器自带无线功能（WiFi），可实现无线组网。

- **交换机**：扩展内网端口，当路由器端口不足时，通过交换机连接多台有线设备（电脑、打印机等），实现有线设备之间的高速通信，不具备连接外网的功能。

- **传输介质**：分为有线和无线两种。有线介质（网线，常用超五类、六类线）传输稳定、速度快，适合固定设备；无线介质（WiFi，支持2.4G/5G频段）灵活便捷，适合移动设备（手机、平板）。

- **终端设备**：需要接入网络的设备，如电脑、手机、打印机、监控摄像头、服务器等。

- **辅助设备**：水晶头、网线钳（制作网线用）、网线测试仪（检测网线连通性）、AP（无线接入点，扩大WiFi覆盖范围）。

## 三、常见局域组网方式（按场景分类）

### 1. 家庭/小型办公组网（1-10台设备）

最常用、最简单的组网方式，无需复杂配置，适合日常使用。

- 组网方案：光猫 → 路由器 → 终端设备（有线/无线），若设备较多，可在路由器后连接交换机扩展端口。

- 优势：成本低、配置简单，插电即可使用，满足日常上网、文件共享需求。

- 注意：路由器需设置WiFi名称、密码，以及登录密码（防止他人蹭网），建议开启加密模式（WPA2/WPA3）。

### 2. 中型办公组网（10-50台设备）

需考虑设备扩展、网络稳定和简单管理，适合中小企业、工作室。

- 组网方案：光猫 → 企业级路由器（支持多端口、限速、防火墙） → 交换机（分层部署，核心交换机+接入交换机） → 终端设备。

- 核心配置：设置静态IP地址（部分设备如服务器、打印机，避免IP变动导致无法访问），划分VLAN（隔离不同部门网络，提升安全性和稳定性）。

- 优势：可管理性强，支持设备扩展，网络稳定，能满足办公设备、监控、服务器等多设备同时运行。

### 3. 大型企业/校园组网（50台以上设备）

需专业规划，注重网络性能、安全性和可扩展性，涉及核心层、汇聚层、接入层三层架构。

- 组网架构：核心层（核心交换机，负责全网数据转发）→ 汇聚层（汇聚交换机，连接核心层和接入层，汇总数据）→ 接入层（接入交换机，连接终端设备），搭配企业级路由器、防火墙、无线AP（全覆盖）。

- 核心配置：部署DHCP服务器（自动分配IP地址）、DNS服务器（解析域名）、防火墙（抵御网络攻击），划分多个VLAN，实现精细化管理。

- 注意：需专业人员调试，定期维护网络，确保网络带宽充足、无卡顿、无安全隐患。

## 四、局域组网基本步骤（以家庭/小型办公为例）

1. **准备设备**：确认光猫（运营商提供）、路由器、网线、终端设备齐全，制作好网线（水晶头与网线连接牢固，可通过测试仪检测）。

2. **连接设备**：将光猫的LAN口用网线连接到路由器的WAN口；若有交换机，将路由器的LAN口连接到交换机的上行口，再将终端设备（电脑、打印机）用网线连接到交换机的LAN口；无线设备可直接搜索路由器WiFi。

3. **配置路由器**：


    - 电脑连接路由器（有线/无线），打开浏览器，输入路由器背面的管理地址（如192.168.1.1），输入默认账号密码（背面标注）。
    
    - 设置上网方式：选择“PPPoE”（家庭宽带常用），输入运营商提供的宽带账号和密码，保存设置，路由器即可连接外网。
    
    - 设置WiFi：修改WiFi名称（SSID）和密码，选择加密模式（WPA2/WPA3），可根据需求设置2.4G和5G双频段（2.4G覆盖广，5G速度快）。
    
    - 修改路由器管理密码：替换默认密码，提升安全性，避免他人篡改配置。

4. **测试网络**：所有设备连接网络后，打开浏览器访问网页，测试上网是否正常；测试设备间通信（如电脑之间共享文件），确认组网成功。

## 五、组网常见问题及解决方法

- 问题1：设备无法连接网络 → 检查网线是否插紧、水晶头是否损坏；路由器WAN口是否连接正确；重启路由器和光猫；确认WiFi密码输入正确。

- 问题2：网络卡顿、速度慢 → 检查路由器是否过载（连接设备过多），可限制单设备带宽；更换六类网线，减少有线传输损耗；调整WiFi频段，避免信号干扰（远离家电、其他WiFi）。

- 问题3：设备间无法共享文件 → 确认所有设备在同一局域网（同一IP段）；开启电脑文件共享功能，设置共享文件夹；关闭电脑防火墙（或添加局域网例外）。

- 问题4：WiFi覆盖范围不足 → 增加无线AP，与路由器组网；调整路由器摆放位置（远离遮挡、居中放置）；更换信号更强的路由器。

## 六、注意事项

- 网络安全：定期修改路由器管理密码和WiFi密码，开启防火墙，避免陌生设备接入；重要数据做好备份，防止局域网内病毒传播。

- 设备选择：根据设备数量、带宽需求选择合适的路由器和交换机，避免设备性能不足导致网络卡顿。

- 布线规范：有线组网时，网线避免与电线、大功率设备靠近，减少信号干扰；网线长度不宜过长（六类线建议不超过100米）。

- 后期维护：定期重启路由器（每月1-2次），清理路由器缓存；及时更新路由器固件，修复安全漏洞。

## 七、补充知识点

1. IP地址：局域网内设备的唯一标识，分为静态IP（手动设置，固定不变）和动态IP（路由器DHCP自动分配，可变动），同一局域网内IP地址不可重复。

2. VLAN：虚拟局域网，将一个物理的局域网在逻辑上划分成多个不同的广播域（子网），不同VLAN之间无法直接通信，需通过路由器转发，提升网络安全性和管理效率。

3. 带宽：局域网有线带宽通常为1000Mbps（千兆），无线带宽根据路由器型号而定，5G频段可达到千兆以上，满足高清视频、大型文件传输需求。

## 八、补充工具：Tailscale 介绍

Tailscale 是一款基于 WireGuard 协议开发的零配置虚拟专用网络（VPN）工具，核心作用是将分散在不同地理位置的设备，虚拟成一个“跨地域的局域网”，让异地设备实现像本地局域网一样的互联互通，无需复杂的网关配置或端口映射，广泛应用于远程办公、异地设备管理等场景，目前已服务超过10,000家组织，包括SAP、Hugging Face等知名企业。

### 1. 核心特点

- **零配置上手**：无需专业网络知识，设备安装客户端后，通过账号登录即可自动接入虚拟网络，自动完成密钥管理和设备发现，摆脱传统VPN复杂的配置流程。

- **P2P直连优先**：采用点对点直连技术，异地设备优先直接传输数据，显著降低网络延迟，若遇到对称NAT、防火墙严格等无法直连的情况，会通过官方DERP中继服务器转发，保证连接稳定性，P2P直连延迟可低至25ms左右。

- **安全加密**：全程采用端到端加密，数据传输过程中无法被窃听或篡改，同时支持访问控制列表（ACL）、双因素认证等安全功能，可精细化控制设备访问权限，保护网络安全。

- **跨设备跨平台**：支持Windows、macOS、Linux、iOS、Android等主流系统，可实现电脑、手机、服务器、NAS等多种设备的无缝组网，甚至能通过子网路由功能，让接入设备访问整个本地局域网的资源。

- **可优化扩展**：支持对等中继功能，可将自身已接入的设备设为中继服务器，提升网络吞吐量；也可自建DERP服务器，解决官方中继服务器在海外导致的国内访问速度慢问题，进一步优化连接性能。

### 2. 与传统局域组网的区别

- **地域限制**：传统局域网受物理范围限制（通常1km内），需依赖本地路由器、交换机等设备；Tailscale 无地域限制，只要设备能接入互联网，无论身处何地，都能加入虚拟局域网。

- **配置难度**：传统组网需手动连接设备、配置路由器（如IP分配、端口映射），复杂场景需专业人员调试；Tailscale 自动完成组网配置，无需手动设置端口转发，防火墙可保持关闭状态，降低使用门槛。

- **核心用途**：传统局域网侧重本地设备互联互通（如家庭、办公内网）；Tailscale 侧重异地设备互联（如远程访问家中NAS、异地服务器联调、分布式团队协作），可作为传统局域网的补充，解决跨地域访问需求。

### 3. 基础使用步骤（简化版）

1. 在需要组网的设备上，安装对应系统的Tailscale客户端（可通过官网下载，Linux系统可通过命令行快速安装）；

2. 启动客户端后，使用GitHub、Google等账号登录，首次登录需在浏览器完成授权，授权后设备自动加入虚拟网络，并获得一个专属的100.x.x.x段虚拟内网IP；

3. 重复步骤1-2，将所有需要互联的设备加入同一Tailscale账号，设备之间可直接通过虚拟IP ping通、访问共享资源，无需额外配置；

4. （可选）若需优化连接性能，可开启对等中继功能，或自建DERP服务器，测试并选择延迟最低的中继节点，提升传输速度。

### 4. 适用场景与注意事项

- **适用场景**：远程访问家中NAS、树莓派等设备；异地服务器之间的内网同步与联调；分布式团队远程协作，安全访问内部服务；公共WiFi环境下，通过加密连接保护设备通信安全，避免数据泄露。
