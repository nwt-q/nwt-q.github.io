# 🧠 终端代理配置笔记（全平台）

------

## 🪟 Windows（CMD / PowerShell）

### 📌 临时代理（当前窗口有效）

```bash
set http_proxy=http://127.0.0.1:7897
set https_proxy=http://127.0.0.1:7897
```

👉 PowerShell：

```powershell
$env:http_proxy="http://127.0.0.1:7897"
$env:https_proxy="http://127.0.0.1:7897"
```

------

### 📌 永久代理（用户级）

```bash
setx http_proxy http://127.0.0.1:7897
setx https_proxy http://127.0.0.1:7897
```

⚠️ 注意：

- 需要**重新打开终端**才生效
- 只影响使用环境变量的程序

------

### 📌 WinHTTP（系统级）

```bash
netsh winhttp set proxy 127.0.0.1:7897
```

查看：

```bash
netsh winhttp show proxy
```

------

## 🍎 macOS / 🐧 Linux

### 📌 临时代理

```bash
export http_proxy=http://127.0.0.1:7897
export https_proxy=http://127.0.0.1:7897
```

------

### 📌 永久代理

写入：

- `~/.bashrc`
- `~/.zshrc`

```bash
export http_proxy=http://127.0.0.1:7897
export https_proxy=http://127.0.0.1:7897
```

生效：

```bash
source ~/.zshrc
```

## 📦 常见开发工具代理配置

------

### 🟢 Git


```bash
git config --global --unset http.proxy
```

# ⚠️ 常见问题总结（重点）

------

## ❌ 1. ping 不通 ≠ 代理失败

```bash
ping google.com
```

👉 **原因：**

- ping 用的是 ICMP
- HTTP 代理不会代理 ICMP

✅ 正确测试：

```bash
curl google.com
```

------

## ❌ 2. 设置了代理但浏览器没用

👉 原因：

- 环境变量只对终端程序有效

✅ 解决：

- 开启代理软件的 **System Proxy**
- 或开启 **TUN 模式（推荐）**

------

## ❌ 3. 端口写错

👉 常见错误：

```bash
127.0.0.1:7897 ❌（实际是7890）
```

✅ 一定要去代理软件确认端口

------

## ❌ 4. 只设置 http 没设置 https

```bash
http_proxy ✔
https_proxy ❌
```

👉 很多请求是 HTTPS → 会失败

------

## ❌ 5. DNS 污染问题

表现：

```bash
ping 能解析 IP，但访问失败
```

✅ 解决：

- 开启 TUN 模式
- 或代理软件开启 DNS 代理

------

## ❌ 6. 代理只对部分工具生效

👉 比如：

- npm ✅
- git ❌

👉 原因：

- 每个工具可能有**独立代理配置**

------

## ❌ 7. Windows 特殊坑

```bash
netsh winhttp show proxy
```

👉 发现：

```
Direct access
```

说明：

👉 系统服务（如某些程序）**没走代理**

------

## ❌ 8. 代理软件没开 / 崩了

👉 很常见：

- 端口存在，但服务没运行
- 配置错误

------

# 🧩 最佳实践（推荐）

------

## ✅ 日常开发推荐组合

👉 打开代理软件：

- 开启 **TUN 模式（最稳）**

👉 然后：

- 不用设置环境变量也能用
- 浏览器 / git / npm 全通

------

## ✅ 只在终端用代理

```bash
export http_proxy=...
```

👉 适合：

- CI
- 服务器
- 临时调试

------

# 🧠 一句话总结

👉 **代理分三层：**

1. 应用层（npm / git）
2. 系统层（System Proxy）
3. 网络层（TUN / VPN）

👉 你现在的问题本质是：

> ❌ 只配了应用层 → 但用系统工具测试（ping）
