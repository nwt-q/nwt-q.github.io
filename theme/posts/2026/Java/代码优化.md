

原始代码

```java
  @PostMapping("/shortlink")
    public ResultResponse wxcode_to_shortlink(@RequestBody QrCodeRequestVO qrcode) {

        /// 校验数据
        if (qrcode == null || qrcode.getAppid() == null || qrcode.getPagepath() == null) {
            return new ResultResponse();
        }

        
        Map<String, Object> Response = From_the_Data(qrcode);

        ResultResponse baseResponse = new ResultResponse();
        baseResponse.setData(Response);
        // 7. 返回数据
        return baseResponse;
    }
```



缺点： **先创建变量，紧接着只设置一个属性**



**解决方案**

针对这种情况，有几种不同程度的简化方案，取决于你是否需要后续操作 `baseResponse` 对象，以及你使用的工具库：

### 1. 直接内联（最推荐）

如果你创建 `baseResponse` 后，紧接着就将其返回或传给下一个方法，而不再修改它，那么完全可以省略中间变量。

```java
// 一行搞定，直接作为返回值或参数
return new ResultResponse(From_the_Data(qrcode));
```

**前提**：`ResultResponse` 需要有接受 `data` 参数的构造函数。

### 2. 链式调用 / Fluent API

如果你不想修改 `ResultResponse` 类，或者希望保持代码的可读性，可以将 `setData` 改为返回 `this`，从而实现链式调用。

**修改类定义：**

```java
public class ResultResponse {
    private Object data;
    
    // 修改 setter，返回 this
    public ResultResponse setData(Object data) {
        this.data = data;
        return this;
    }
    // ...
}
```

**简化后的调用：**

```java
// 不需要中间变量
ResultResponse baseResponse = new ResultResponse().setData(From_the_Data(qrcode));
```

### 3. 使用 Lombok (@Builder)

如果你的项目中使用了 Lombok，这是最优雅的方式。通过注解自动生成构建器代码。

**修改类定义：**

```java
import lombok.Builder;
import lombok.Data;

@Data
@Builder
public class ResultResponse {
    private Object data;
}
```

**简化后的调用：**

```java
// 语义非常清晰
ResultResponse baseResponse = ResultResponse.builder()
    .data(From_the_Data(qrcode))
    .build();
```

### 4. 静态工厂方法

如果你不想引入 Lombok，也不想修改 setter，可以在 `ResultResponse` 类中增加一个静态工厂方法。

**修改类定义：**

```java
public class ResultResponse {
    private Object data;

    public static ResultResponse of(Object data) {
        ResultResponse response = new ResultResponse();
        response.setData(data);
        return response;
    }
    // ...
}
```

**简化后的调用：**

```java
ResultResponse baseResponse = ResultResponse.of(From_the_Data(qrcode));
```

### 📌 总结建议

- **最省事**：如果只是为了返回结果，直接 `return new ResultResponse(data);`。
- **最现代**：如果有 Lombok，使用 `@Builder`。
- **最通用**：如果没有 Lombok，添加一个静态工厂方法 `of()` 是很好的实践。
